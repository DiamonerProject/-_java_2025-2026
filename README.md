Secure File Transfer (Client–Server)

Описание проекта

Данный программный комплекс реализует защищенную передачу файлов в незащищенной сетевой среде с использованием
криптографического шифрования и цифровой подписи.  
Система построена по клиент-серверной архитектуре и разработана в рамках курсовой работы по дисциплине
«Информационная безопасность».

Цель проекта — обеспечить конфиденциальность, целостность и аутентичность передаваемых данных при обмене файлами
между клиентом и сервером.

---

Используемые технологии

- Язык программирования: Java
- Графический интерфейс: Swing
- Сетевое взаимодействие: Socket API
- Криптография: Java Cryptography Architecture (JCA), Java Cryptography Extension (JCE)

---

Реализованные криптографические механизмы

- Асимметричное шифрование: RSA
- Алгоритм подписи: SHA256withRSA
- Размер ключа: 2048 бит
- Шифрование выполняется с использованием открытого ключа сервера
- Цифровая подпись формируется закрытым ключом клиента

---

Структура проекта

src/

client/

Подготовка к запуску

 Требования
- Java JDK версии 8 или выше
- Любая IDE (IntelliJ IDEA, Eclipse, NetBeans) или командная строка

---

Порядок запуска

 1. Запуск сервера
1. Запустить класс `ServerGUI`
2. Указать порт (по умолчанию 8080)
3. Нажать кнопку Start Server
4. При первом запуске автоматически генерируются ключи сервера

 2. Запуск клиента
1. Запустить класс `ClientGUI`
2. Выбрать файл для передачи
3. Указать адрес сервера и порт
4. Нажать кнопку Send File
5. При первом запуске автоматически генерируются ключи клиента

---

 ВАЖНО

Для корректной работы системы необходимо, чтобы:
- клиент имел доступ к открытому ключу сервера (`server_public.key`)
- сервер имел доступ к открытому ключу клиента (`client_public.key`)

Открытые ключи должны быть скопированы между сторонами вручную.

Примеры работы

 Положительный сценарий
- Файл успешно зашифрован
- Цифровая подпись корректна
- Сервер расшифровывает данные и сохраняет файл

 Негативный сценарий
- Использование неверного открытого ключа клиента
- Проверка цифровой подписи завершается ошибкой
- Файл отклоняется сервером

---

Ограничения реализации

- Используется RSA для шифрования файла целиком (ограничение по размеру данных)
- Отсутствует симметричное шифрование и гибридная схема
- Алгоритмы выбираются на этапе конфигурации, без динамического переключения
